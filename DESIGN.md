# negaflow 디자인 시스템

## 1. 분위기와 정체성

negaflow는 사진 자체가 중심이고 조작부는 조용히 물러나는 macOS 필름 현상실입니다. 시그니처는 캔버스를 넓게 남기고, 필요한 순간에만 반투명 도구와 인스펙터가 읽히는 "사진 우선의 유리 표면"입니다.

## 2. 색상

| 역할 | 토큰 | 사용 |
|---|---|---|
| 기본 표면 | `surface.canvas` | 시스템 창 배경과 중앙 캔버스 |
| 보조 표면 | `surface.material` | `regularMaterial` 기반 사이드바와 도구 표면 |
| 기본 텍스트 | `text.primary` | `Color.primary` |
| 보조 텍스트 | `text.secondary` | `Color.secondary` |
| 동작 강조 | `accent.action` | `Color.accentColor`, 선택·확정 동작만 |
| 경고 | `status.warning` | 스캐너 대기·주의 상태 |
| 오류 | `status.error` | 실패·취소 같은 파괴적 동작 |

규칙: 장식용 강조색과 그라데이션을 사용하지 않습니다. 색상은 macOS의 동적 시스템 색상과 소재에 맡기고, 강조색은 실제 조작 가능 상태에만 사용합니다.

## 3. 타이포그래피

| 단계 | SwiftUI 스타일 | 용도 |
|---|---|---|
| 창 제목 | `.headline` | 앱 이름, 핵심 작업 제목 |
| 섹션 | `.subheadline.weight(.semibold)` | Scan, Base, 현상 섹션 헤더 |
| 본문 | `.body` | 선택지와 설명 |
| 보조 | `.caption` | 제어 레이블 |
| 메타데이터 | `.caption2.monospacedDigit()` | 프레임 번호, 값, DPI |

글꼴은 SF 계열 시스템 글꼴만 사용합니다. 수치와 해상도 표기는 고정폭 숫자로 흔들림 없이 정렬합니다.

## 4. 간격과 레이아웃

기본 단위는 4pt입니다. 인라인 간격은 8pt, 제어 묶음은 12pt, 패널 내부 여백은 14-16pt를 사용합니다. 좌측 프레임 목록은 기본 180pt, 우측 인스펙터는 기본 252pt로 두고 중앙 캔버스를 우선합니다. 앱 최소 너비에서는 중앙 캔버스가 560pt 이상 유지되어야 합니다.

## 5. 구성요소

### 3열 작업 공간
- **구조**: 프레임 목록, 중앙 캔버스, 현상 인스펙터.
- **상태**: 선택 없음, 스캔 중, 현상 중, 프레임 선택.
- **접근성**: 모든 아이콘은 접근성 레이블과 도구 설명을 가집니다.

### 워크플로 섹션
- **구조**: 전체 폭 헤더, 펼침 콘텐츠, 구분선.
- **상태**: 닫힘, 열림, 처리 중 비활성화.
- **동작**: 헤더 어디를 눌러도 열고 닫습니다. 초기화는 해당 섹션의 수동 조절만 되돌립니다.

### 도구 스트립
- **구조**: 크롭, 회전, 좌우·상하 반전, 변형 초기화 아이콘.
- **상태**: 활성 변형, 비활성 초기화.
- **동작**: 회전·반전은 현재 프레임과 이 세션의 이후 스캔에 함께 적용하고, 크롭은 현재 프레임에만 적용합니다.

## 6. 모션과 상호작용

마이크로 상호작용과 섹션 전환은 180ms 내외의 `.snappy` 애니메이션을 사용합니다. 크롭·현상·스캔이 실행 중일 때는 충돌하는 제어를 비활성화하고, 진행 상태를 숨기지 않습니다. 상호작용 가능한 도구 표면만 Liquid Glass의 interactive 변형을 사용합니다.

## 7. 깊이와 표면

깊이 전략은 macOS 소재 기반의 톤 차이입니다. macOS 26 이상에서는 인접한 아이콘 도구를 `GlassEffectContainer` 안에서 렌더링하고, 그 이전 버전에서는 `regularMaterial`을 사용합니다. 그림자, 두꺼운 카드 테두리, 장식성 블러는 사용하지 않습니다.
